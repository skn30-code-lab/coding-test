# 🚀 Coding Test Study

매일 꾸준히 문제를 풀고, 서로의 코드를 공유하며 함께 성장하는 코딩 테스트 스터디입니다.

---

## ⏰ 운영 방식
- **모임 요일**: 월 / 수
  
- **시간**: 19:30 ~ 20:30 (최대 1시간)

- **진행 방식**
  - 정해진 문제를 각자 풀이합니다.
  - 풀지 못한 경우, 막힌 부분을 공유하고 함께 해결하는 시간을 갖습니다.
    - ("지금 이 부분이 잘 안 되는데, 도와주실 수 있을까요?")

- **불참 시**
  - 참여하지 못할 경우, 설명 주석이 달린 코드를 해당 날짜 폴더에 미리 업로드합니다.

---

## 📂 폴더 및 파일 규칙

문제별로 폴더를 생성하고, 해당 폴더 안에 본인의 파일을 업로드합니다.

### 1. 폴더명 규칙

- **백준**: `YYMMDD_BJ_문제이름`  
  (예: `260402_BJ_설탕배달`)

- **프로그래머스**: `YYMMDD_PMS_문제이름`  
  (예: `260402_PMS_입국심사`)

---

### 2. 파일명 규칙

```
solution_이름.py
solution_이름.ipynb
```

예시:
`solution_jexists.py`


---

## 💬 커밋 컨벤션 (Commit Convention)

### 형식

prefix: [이름] 문제번호_문제이름


### Prefix 의미 및 사용 상황

| Prefix   | 의미       | 사용 상황 |
|----------|------------|----------|
| solve    | 문제 해결  | 문제를 완벽히 풀어서 처음 제출할 때 |
| wip      | 진행 중    | 풀이 중인 코드를 중간 저장하거나 질문을 위해 올릴 때 |
| fix      | 수정       | 이미 올린 코드의 오류를 고치거나 로직을 수정했을 때 |
| refactor | 리팩토링   | 기능 변화 없이 코드를 효율적으로 개선했을 때 |
| rename   | 이름 변경  | 파일명이나 폴더명을 규칙에 맞게 수정했을 때 |
| remove   | 삭제       | 불필요한 파일을 지울 때 |

## 🛠 참여 방법 (Workflow)

### 초기 설정 (최초 1회)

```bash
git config --global user.name "본인이름"
git config --global user.email "이메일주소"
git clone [Organization_URL]
```

### 데일리 루틴
```
# 1. 최신 상태 동기화
git pull origin main

# 2. 파일 작업 후 스테이징 & 커밋
git add .
git commit -m "solve: [이름] 문제번호_문제이름"

# 3. 서버에 반영
git push origin main
```

## 스터디 목적

이 스터디는 단순 문제풀이를 넘어서, 아래 목표를 가지고 진행됩니다.

:one: PCCE 취득을 목표로 합니다.
→ 시험 대비를 통해 코딩테스트 기본기를 확실히 다집니다.

:two: 알고리즘에 대한 이해를 목표로 합니다.
→ 단순 암기가 아니라, “왜 이렇게 푸는지”를 이해하는 데 집중합니다.

:three: 궁극적으로 좋은 개발자가 되는 것을 목표로 합니다.
→ 문제 해결력과 사고력을 키워 실무에서도 통하는 개발자로 성장합니다.

:fire: 꾸준히 참여하는 것이 가장 중요합니다! 함께 성장해봅시다 :muscle:
